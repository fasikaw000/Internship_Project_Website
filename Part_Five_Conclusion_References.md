   - Study the basics of web development (HTML, CSS, JavaScript)

2. **Research the Company**
   - Learn about the host organization's business model and services
   - Understand the technologies they commonly use
   - Review their portfolio or previous projects
   - Prepare relevant questions for your supervisors

3. **Set Clear Goals**
   - Define what you want to learn during the internship
   - Identify specific skills you want to develop
   - Set measurable objectives for your internship period

5. **Embrace Challenges**
   - Don't be afraid to tackle difficult problems

#### After the Internship

2. **Maintain Connections**

### Appendix B: Database Schema Diagrams

#### User Collection Schema
```javascript
{
  _id: ObjectId,
  fullName: String,
  username: String (unique),
  email: String (unique),
  password: String (hashed),
  role: String ["user", "admin"],
  accountNumber: String (optional, admin only),
  createdAt: Date (auto-generated)
}
```

#### Product Collection Schema
```javascript
{
  _id: ObjectId,
  name: String,
  category: String ["electronics", "fashions", "books", "all"],
  price: Number,
  description: String,
  image: String (file path),
  stock: Number,
  createdAt: Date (auto-generated)
}
```

#### Order Collection Schema
```javascript
{
  _id: ObjectId,
  user: ObjectId (ref: User),
  products: [{
    product: ObjectId (ref: Product),
    quantity: Number
  }],
  totalPrice: Number,
  deliveryInfo: {
    name: String,
    phone: String,
    email: String,
    address: String
  },
  receiptImage: String (file path),
  status: String ["pending", "verified", "delivered", "cancelled"],
  createdAt: Date (auto-generated)
}
```

#### Comment Collection Schema
```javascript
{
  _id: ObjectId,
  product: ObjectId (ref: Product),
  user: ObjectId (ref: User),
  text: String,
  createdAt: Date (auto-generated)
}
```

**API (Application Programming Interface)**: A set of protocols and tools for building software applications.

**Authentication**: The process of verifying the identity of a user.

**Authorization**: The process of determining what actions a user is allowed to perform.

**Backend**: The server-side of an application that handles business logic and data management.

**CRUD**: Create, Read, Update, Delete - the four basic operations of persistent storage.

**Frontend**: The client-side of an application that users interact with directly.

**JWT (JSON Web Token)**: A compact, URL-safe means of representing claims to be transferred between two parties.

**Middleware**: Software that acts as a bridge between an operating system or database and applications.

**MongoDB**: A NoSQL document-oriented database program.

**Node.js**: A JavaScript runtime built on Chrome's V8 JavaScript engine.

**React**: A JavaScript library for building user interfaces.

**REST (Representational State Transfer)**: An architectural style for designing networked applications.

**Schema**: The structure that defines the organization of data in a database.

**TailwindCSS**: A utility-first CSS framework for rapidly building custom user interfaces.

**Vite**: A modern frontend build tool that provides a faster development experience.
